Burnmeter

Track your infrastructure burn rate across 11 platforms in one place

Get Started on GitHub

The Problem

You're running on Vercel, OpenRouter, Anthropic, Neon, and five other platforms. Each has its own billing dashboard. You have no idea what you're actually spending.

The Solution

Burnmeter is an MCP server that aggregates costs across all your platforms. Ask Claude "what's my burn this month?" and get a real answer.

Quick Start

1 Install

git clone https://github.com/mpalermiti/burnmeter.git
cd burnmeter
python3 -m venv venv
source venv/bin/activate  # or `venv\Scripts\activate` on Windows
pip install -e .

2 Configure

cp .env.example .env
# Add your API keys to .env

3 Add to Claude

Add to your ~/.claude/mcp_config.json:

{
  "mcpServers": {
    "burnmeter": {
      "command": "/path/to/burnmeter/venv/bin/python",
      "args": ["/path/to/burnmeter/src/server.py"],
      "env": {
        "PYTHONPATH": "/path/to/burnmeter/src"
      }
    }
  }
}

4 Use with Claude

You: "What's my total infrastructure spend this month?"
Claude: *uses burnmeter to aggregate across all platforms*

Features

Multi-platform aggregation – See total costs across all services in one query

Budget monitoring – Check if you're over budget before the bill arrives

Platform breakdowns – Drill into specific platforms for detailed cost data

Intelligent caching – 1-hour TTL to respect API rate limits

Open source – Apache 2.0 license, contribute providers for your stack
